1. 게임 개요
----------------------------------------------------------------
- 장르: 3D 턴제 전술 전략 게임
- 엔진: Three.js r160 (ES Modules, CDN importmap)
- 플랫폼: 웹 브라우저 (index.html 단일 파일 진입)
- 개발 경로: d:\KwangwoonUniv\DAKER\GAME\tactical\

플레이어는 최대 4명의 아군 캐릭터를 7x9 그리드 위에 배치하고,
웨이브마다 밀려오는 적을 처치하며 보스를 쓰러뜨리는 것이 목표다.
적을 처치할수록 티켓을 획득하고, 티켓으로 캐릭터를 강화하거나
새 슬롯을 해금하는 성장 루프를 갖는다.

3. 그리드 시스템
----------------------------------------------------------------
- 크기: 7열 x 9행 (COLS=7, ROWS=9)
- 셀 크기: CELL = 1.4 (Three.js world units)
- 좌표계: gridX = 열(0~6), gridZ = 행(0~8)
  - 행 0~2: 아군 배치 구역 (녹색 틴트)
  - 행 3~5: 중립 구역
  - 행 6~8: 적 배치 구역 (적색 틴트)
- toWorld(col, row) -> Vector3 (중앙 기준 오프셋)
- fromWorld(wx, wz) -> {col, row} | null
- cells[c][r] = { mesh, occupied: null | Character | Enemy }

4. 캐릭터 시스템
----------------------------------------------------------------

[아군 3종]

전사 (attacker)
- 아이콘: 검
- 색상: 붉은색
- 공격 범위: 원형 (Chebyshev 거리, 기본 1칸)
- 기본 공격력: 30, 레벨당 +15
- 특징: 주변 8칸 광역 근접 공격

성직자 (healer)
- 아이콘: 별
- 색상: 녹색
- 공격 범위: 원형 (기본 2칸)
- 회복량: 25, 레벨당 +15
- 특징: 범위 내 아군 체력 회복 (자신 제외)

궁수 (piercer)
- 아이콘: 활
- 색상: 파란색
- 공격 범위: 직선 (같은 열, 앞뒤 기본 1칸)
- 기본 공격력: 20, 레벨당 +10
- 특징: 같은 열 양방향 관통 공격

[공통 스탯]
- 기본 HP: 80
- 레벨업 시 maxHp += 20, 현재 HP +30 회복
- 레벨 범위: 1~4 (fragment 3개 = 1레벨업)

[범위 계산]
- 원형: max(|dx|, |dz|) <= range  (Chebyshev 거리, 정사각형 범위)
- 직선: ax == gx && |az - gz| <= range  (같은 열, 앞뒤 대칭)
- range = baseRange + (level - 1) * rangePerLevel  (정수 셀 단위)
- 시각화 반경 = range * CELL  (world units)

6. 전투 시스템
----------------------------------------------------------------

[아군 턴]
1. 캐릭터 클릭 -> BFS로 이동 가능 칸 강조 (이동 범위: 2칸)
2. 대상 칸 클릭 -> 이동 + 즉시 공격 실행
3. 모든 아군이 이동하면 자동으로 적 턴 전환
   (또는 [턴 종료] 버튼으로 수동 종료)

[BFS 이동 제한]
- 적이 점유한 칸은 통과 불가
- 아군이 점유한 칸은 통과 가능 (도착 불가)
- 이동 범위 2칸 (상하좌우 4방향)

[공격 처리 - applyAttack()]
- 이동 완료 후 현재 위치 기준으로 getTargets() 호출
- 힐러(baseHeal>0)면 아군만 회복, 나머지는 적 공격
- 사망 처리: 아군 -> returnToRoster(), 적 -> scene 제거 + 티켓 획득

[적 턴 - resolveEnemyTurn()]
- 모든 적이 순서대로 타입별 AI 실행
- 700ms 딜레이 후 시작, 400ms 딜레이 후 플레이어 턴 복귀

[사망/부활 처리]
- 아군 사망 시 roster 대기 상태로 복귀 (placed=false)
- 메시 제거, HP 전체 회복
- 티켓 1장 소모하여 재배치 가능

[게임 종료 조건]
- 패배: 모든 아군이 roster에서 대기 중이고 배치 불가 상태
- 승리: 보스 처치 (현재 미구현, 웨이브 계속 진행)

7. 성장/강화 시스템
----------------------------------------------------------------

[티켓]
- 적 처치 시 획득 (적 타입별 보상량 상이)
- 드로우, 재배치, 해금에 사용

[드로우 (티켓 1장)]
- randomType()으로 랜덤 타입 조각 1개 획득
- fragments[type][level] 카운터 증가
- 3개 모이면 자동 강화 triggerUpgrade() 호출

[강화 (triggerUpgrade)]
- 해당 타입 배치 중인 캐릭터 레벨업
- 없으면 roster 대기 캐릭터 레벨업
- 그것도 없으면 해당 레벨의 새 캐릭터 roster 추가

[해금 (티켓 3장)]
- 전체 roster 보유 캐릭터 수가 4미만일 때 사용 가능
- 랜덤 타입 새 캐릭터 1명 roster에 추가
- 4번째 배치 슬롯 개방 효과
- 1회성 (보유 4명이 되면 버튼 비활성화)

[재배치 (티켓 1장)]
- 사망 후 roster 대기 중인 캐릭터를 다시 필드에 배치
- 강화카드 UI의 [배치] 버튼 클릭 후 빈 칸 클릭

[액션 버튼 - 하단 우측]
- [해금 🎟️3]: 4번째 슬롯 해금 (보라색)

- [턴 종료]: 수동 턴 넘기기 (금색)

[범위 시각화]
- 아군 선택 시: 이동 대상 칸에 hover하면 공격 범위 링 미리보기

- 적 hover 시: 빨간 공격 범위 링 표시

[레디 링]
- 아직 이동하지 않은 아군 발 아래 청록 링 맥박 애니메이션
- 이동 완료 시 사라짐

[에러 오버레이]

11. 주요 버그 수정 이력
----------------------------------------------------------------
- rangeCells is not defined -> clearSelection()에서 미사용 변수 제거
- 빈 화면 -> PointLight position.set() 방식으로 수정
- OrbitControls가 캐릭터 클릭 차단 -> pointerdown/up 분리 처리
- 궁수 데미지 미적용 -> world 단위 범위를 정수 셀 단위로 전환
- 궁수 단방향만 공격 -> |az - gz| <= range 양방향 조건으로 수정
- 대각선 공격 누락 -> Chebyshev 거리(max(|dx|,|dz|))로 전환
